## Task Description
Verify that the validator loads real track JSON from `/public/tracks/` instead of using a hardcoded straight-line fixture.

## Finding: Implementation Already Complete

The track loading functionality was already implemented in a prior commit. The validator correctly loads real track data from JSON files.

### Implementation Details

1. **TrackStore Module** (`crates/validator/src/track.rs`)
   - Loads track JSON files from a directory (defaults to `apps/web/public/tracks`)
   - Parses terrain, obstacles, surfaces, ramps, hazards
   - Converts to WASM ABI format for the re-simulation engine

2. **Main Validator** (`crates/validator/src/main.rs`)
   - Lines 39-58: TrackStore initialization from TRACKS_DIR env var
   - Lines 344-353: Real track data retrieval by track_id
   - Lines 374-383: Real terrain and obstacles passed to resim engine

3. **Track JSON Format** (e.g., `apps/web/public/tracks/hills-01.json`)
   - Contains terrain points, obstacles, surfaces, ramps, hazards
   - Used by both client and validator for consistency
