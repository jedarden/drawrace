# Wheel-Shape Progression Constraints (BF-5R57)

**Status:** Already implemented in commit 8816442

## Summary

Optional wheel-shape constraints as progression hooks for post-v1 gameplay:
- **Single-stroke mode**: Player must draw wheel in one continuous stroke
- **Convex-only mode**: Only convex (round) shapes allowed; concave shapes rejected

## Implementation Details

### Engine Core (`packages/engine-core/src/draw-pipeline.ts`)
- `DrawConstraints` interface: `{ singleStroke?: boolean; convexOnly?: boolean; }`
- `ConstraintViolation` type: `{ type: "single-stroke" | "convex-only"; message: string; }`
- `validateConstraints()` function: checks stroke count and convex decomposition result

### UI Components

**DrawScreen** (`apps/web/src/DrawScreen.tsx`):
- Tracks stroke count (`strokeCountRef`)
- Validates on "Race!" button press
- Shows active constraint badges (e.g., "Single-Stroke", "Convex-Only")
- Displays constraint violation messages with visual feedback

**DrawOverlay** (`apps/web/src/DrawOverlay.tsx`):
- Validates mid-race redraws against constraints
- Silently rejects invalid strokes (no alert needed during race)

**SettingsScreen** (`apps/web/src/SettingsScreen.tsx`):
- "Challenge Modes" section with toggle switches
- Persists to `localStorage` key `drawrace.constraints`
- Notifies parent of constraint changes

### State Management (`apps/web/src/App.tsx`)
- Loads constraints from localStorage on init
- Passes constraints through to DrawScreen, RaceScreen, DrawOverlay
- Handles constraint change callbacks from SettingsScreen

### Tests (`packages/engine-core/src/draw-pipeline.test.ts`)
- Comprehensive test coverage for `validateConstraints()`
- Tests for single-stroke, convex-only, and combined constraints
- Tests for violation messages and edge cases

All tests pass (20/20).
